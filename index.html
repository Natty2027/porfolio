Fintech · Bank data

Money Firewall

Subscription-detection web app that connects to a user's bank via Plaid and surfaces recurring outflows from real transaction data. Detection groups outflows by normalized merchant, classifies cadence (weekly / monthly / annual) by median interval, auto-marks cancelled subs when an expected cycle has elapsed, and recommends renegotiate / review / keep actions.

Built as a pnpm monorepo. API contract lives in an OpenAPI spec; Orval codegens React Query hooks and Zod schemas from it so the client and server can never drift. Server has a simulator mode that seeds a 9-merchant transaction history when Plaid credentials are absent — useful for review demos.

Engineering highlights · Cognivate deep-dive

Generation pipeline

Reliable AI generation for variable-quality inputs

Built a generation pipeline that accepts noisy student uploads (scanned PDFs, phone-camera photos of handwritten notes, pasted snippets) and produces consistently-structured study modules. Implemented structured output, retry-on-parse-error, length-aware chunking for long materials, and graceful fallbacks for insufficient input.

Subscription infrastructure

Production payments with RevenueCat + StoreKit

Wired Apple in-app subscriptions through RevenueCat: App Store Connect API key integration for server-to-server receipt validation, entitlement-driven feature gates throughout the app, sandbox testing pipeline, and a hardened paywall that handles purchase cancellation, restoration, and post-expiration UX. Designed for App Review review-flow requirements.

Auth flow

Custom-domain Clerk auth with Google and Apple sign-in

Authentication runs on a custom Clerk domain (clerk.cognivate.co) with OAuth provider passthrough — Google sign-in uses our own OAuth client so Clerk's branding doesn't appear at consent. Native-app SSO deep-link handling, allow-listed redirect URLs, and post-sign-in entitlement sync to RevenueCat.

Release engineering

EAS Build pipeline with parallel-install preview track

Single Expo project ships two app identities: the public TestFlight beta (com.cognivate.mobile) and a v2-preview variant (com.cognivate.mobile.v2) for in-flight features. Selected at build time via APP_VARIANT, so iOS treats them as separate apps and testers can run both side-by-side without overwriting each other.
