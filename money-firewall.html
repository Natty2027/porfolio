Money Firewall

A subscription-detection web app that connects to your bank via Plaid and catches the recurring outflows you forgot about. Renegotiate, review, or cancel — without scrolling through twelve months of statements.

TypeScript Node.js Express 5 PostgreSQL Drizzle ORM Zod Orval Plaid pnpm workspaces

The idea

The average household carries somewhere around twelve recurring subscriptions, and almost nobody remembers them all. Money Firewall connects to your bank, scans your outflows, groups them by merchant, classifies cadence, and tells you which ones look like dead weight.

What I built

  • Detection engine — groups outflows by normalized merchant key (minimum two occurrences), classifies cadence by median interval (weekly / monthly / annual), and auto-marks as cancelled when more than two times the expected interval has elapsed. Recommends "renegotiate" on >8% price drift, "review" on overdue or single-cycle annual, "keep" otherwise. User-set status is preserved on resync.
  • Plaid integration with simulator fallback — production mode opens Plaid Link and exchanges the public token for real account data. If server credentials are absent (review mode, demo), the client falls back to a simulator that seeds a 9-merchant transaction history. Same API contract either way.
  • OpenAPI-driven monorepo — single source of truth in lib/api-spec/openapi.yaml. Orval codegens React Query hooks and Zod schemas from the spec, so the client and server can never drift. Body schemas are operation-named so refactors stay typed.
  • Auth + sessions — email-only login, httpOnly mf_session cookie (30-day rolling), requireUser middleware on protected routers. Frontend Auth provider gates rendering and exposes loginAsync / logoutAsync.
  • Persistence + onboarding gate — Drizzle ORM schema for users, Plaid items, transactions, subscriptions. App routes a new user to /onboarding until hasConnectedBank flips true.

The "wow"

The simulator mode is the part nobody asks for but everyone is glad exists. Whole purchase + detection loop works without real Plaid credentials, which makes review demos and recorded walkthroughs trivially easy.
