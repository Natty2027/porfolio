Cognivate

An AI study app that turns any document into chapters, vocab, quizzes, and interactive practice. Live on the iOS App Store — build 41 on TestFlight.

React Native Expo TypeScript EAS Build Claude API Clerk RevenueCat Sentry react-native-svg React Query Drizzle ORM

The idea

Students upload PDFs, photos of handwritten notes, and pasted text. Cognivate uses Claude to turn that messy input into structured study modules — readable chapters, vocab cards for spaced repetition, and adaptive quiz questions. Plus an audio tutor mode for hands-free review.

What I built

  • Generation pipeline — accepts noisy student uploads and produces consistently-structured study modules. Structured output, retry-on-parse-error, length-aware chunking for long materials, graceful fallbacks for insufficient input. Per-file 25K char and 100K total context window so more source material reaches the AI.
  • Adaptive practice + grader — interactive practice problems with a deterministic application-layer grader. Normalizes thousands-commas, unicode minus signs, fraction notation (1/2 → 0.5), and uses the expected answer's decimal precision to avoid false "nearly correct" grades.
  • Graph Lab — STEM coordinate plotter. AI extracts 3–5 data points and axis metadata from uploaded materials (minimal ~600-token call). User taps an SVG grid to place points; they snap to the nearest grid intersection. Graded exact / within-one-step / incorrect with visual feedback.
  • Source badges — every AI response carries a source-tier badge (From source, AI fill, etc.). Tapping opens a bottom sheet with the actual document excerpt that grounded the answer — file name, section location, and an italic quote from the source text.
  • Subscription infrastructure — Apple in-app subscriptions through RevenueCat with server-to-server receipt validation via App Store Connect API key. Entitlement-driven feature gates, sandbox testing pipeline, and a hardened paywall that handles cancellation, restoration, and post-expiration UX.
  • Custom-domain auth — Clerk on clerk.cognivate.co with custom OAuth credentials so Google's consent screen shows our branding. Native-app SSO deep-link handling, allow-listed redirect URLs, post-sign-in entitlement sync to RevenueCat.
  • EAS Build release pipeline — single Expo project ships two app identities. Public App Store build (com.cognivate.mobile) and a v2-preview variant for in-flight features. Selected at build time via APP_VARIANT so iOS treats them as separate apps and testers can run both side-by-side.

What went wrong (and got fixed)

Apple rejected the first submission with four issues: ATT/tracking labels, a Google sign-in redirect URL mismatch, an authentication-code blocker on the demo account, and a missing expired-subscription demo state. Fix walked through every dashboard: App Store Connect privacy labels, Clerk's mobile SSO allowlist, App Store Connect API key for RevenueCat receipt validation, and a sandbox subscription cycle to demonstrate the expired state.
